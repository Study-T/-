# Product Brief: 3D 数字人虚拟试穿系统

## Executive Summary

为电商平台构建可交付的 3D 数字人虚拟试穿系统。用户上传一张照片即可生成自己的 3D 数字分身，在线试穿服装和鞋子，360° 查看上身效果。系统作为一个完整项目交付给电商平台客户，支持私有化部署，覆盖国内和国际市场，男女品类皆适配。

核心策略是利用和改造开源 AI 模型（LHM、FASHN VTON），以最低的基础设施成本实现可商用的试穿效果。壁垒在于：把多模态 AI 管线（3D 生成 + 2D 试穿 + 3D 渲染）调成一套可交付的产品，以及针对客户场景做深度定制——这两件事竞品不容易复制，因为涉及大量工程细节和场景适配，而非拼模型精度。

## The Problem

线上服装零售面临结构性痛点：
- **退货率高企**：线上退货率达 19.3%（NRF 2025），其中"不合身/效果不符"是首要原因，美国年退货总额达 $850B
- **用户决策盲区**：传统电商只有平面模特图，买家无法感知衣服穿在"自己身上"的效果，"bracketing"（买多件退多件）成为常态
- **平台竞争升级**：Google 已将 AI 试穿集成到搜索结果（2026年4月），Shopify 集成了 Genlook，Zara 2025年底上线试穿——虚拟试穿正从实验变为竞争必需品

中国电商和跨境电商平台对虚拟试穿有强烈需求。跨境场景尤为突出——各国尺码标准不统一，退货率远超 19.3%（部分品类超 40%）。但市面方案要么是 SaaS 按次收费（长期成本不可控），要么是闭源黑盒（无法定制），存在"可私有化部署 + 低成本 + 可定制"的市场空白。

## The Solution

一套完整的 3D 虚拟试穿系统，作为项目交付给客户平台：

1. **3D 数字人生成**：用户上传单张照片 → LHM 模型秒级生成全身 3D 数字人（SMPL-X 骨骼），也支持手动调参微调
2. **AI 虚拟试穿**：用户选择服装商品 → FASHN VTON 将服装迁移到数字人身上 → 360° 旋转查看各角度效果
3. **动画与场景**：Mixamo 预设骨骼动画（走路/转圈/挥手）+ 多场景切换（商场/沙滩/办公室），增强沉浸感
4. **Web 端交付**：基于 Three.js + LAM_WebRender 高斯泼溅渲染，浏览器即开即用，手机端目标 ≥ 25FPS（中端机型）

一次交付，客户私有化部署，无持续 API 调用费用，且源码可定制。

**关键技术挑战**：FASHN VTON 输出 2D 试穿效果图，需通过 UV 纹理映射回 SMPL-X 3D 模型实现 360° 查看。这是整个管线的核心工程难点，也是竞品难以快速复制的关键环节。

## What Makes This Different

| 维度 | 市面方案 | 我们 |
|------|---------|------|
| 交付方式 | SaaS 按调用收费 | **一次性项目交付 + 私有化部署** |
| 模型 | 闭源自研/商业 API | **开源改造 + 工程优化**，不绑特定厂商 |
| 成本结构 | 持续 API 费用，量越大越贵 | GPU 自有，边际成本仅电费和硬件折旧 |
| 可定制性 | 黑盒，难以修改 | 源码交付，客户可二次开发 |
| 品类 | 多聚焦女装上衣 | 男女皆覆盖，上下装+鞋子 |
| 渲染 | 多数 2D 效果图 | **Web 3D 高斯泼溅**，手机浏览器可用 |

核心壁垒不是算法，而是**管线的工程化整合与场景适配**——把 LHM + FASHN VTON + LAM_WebRender + SMPL-X 调通、优化、产品化，并在具体客户场景中深度定制。2D 试穿结果到 3D 模型的纹理映射是关键技术门槛，这套工程经验积累起来就是护城河。

## Who This Serves

**直接客户**：
- **电商平台**：国内垂直/品牌电商平台，以及跨境电商平台（SHEIN/Temu 生态），希望拥有虚拟试穿但不愿绑定 SaaS 订阅
- **品牌直营**：跨境服饰品牌（如安踏、SHEIN 上的大卖家），退货率更高（跨境部分品类退货 > 40%），对转化和降退货诉求更强，决策链路比平台更短

**终端用户**：平台或品牌独立站上的消费者，20-40 岁男女，网购服装频次高，因退货经历有决策焦虑。

**关键使用场景**：
- 买家选衣服 → 点"试穿" → 5 秒看效果 → 决策买不买
- 卖家上新品 → 用统一数字人展示多款服装 → 降低模特拍摄成本

## Success Criteria

| 指标 | 目标 |
|------|------|
| 项目交付客户数（首年） | 3-5 个 |
| 客户平台退货率降低 | ≥ 3 个百分点 |
| 试穿到加购转化率 | ≥ 15% |
| 数字人生成耗时 | < 5 秒 |
| 试穿效果生成耗时 | < 8 秒 |
| 单次试穿 GPU 成本 | < ¥0.10（T4/L4 实测后校准） |
| 手机端渲染帧率 | ≥ 25 FPS |

## Scope

### In Scope (v1)
- 照片数字人生成（LHM）
- 手动参数微调（身高/体重/肩宽/胸围/腰围/臀围/腿长）
- 上装 + 下装 + 鞋子试穿
- 360° 旋转查看 + 缩放
- 预设骨骼动画（4 个）+ 场景（5 个）
- 私有化部署方案 + 运维文档

### Out of Scope (v1)
- AR 实时摄像头试穿
- 多人同时试穿/社交分享
- 短视频生成
- AI 尺码推荐
- 移动端原生 App（浏览器即可）

## Competitive Landscape

| 类型 | 代表 | 弱点 |
|------|------|------|
| 大厂方案 | Google AI Try-On、Amazon | 不卖项目，绑定自身生态 |
| SaaS 方案 | DRESSX、Vue.ai、Zyler | 按次收费，长期成本高 |
| AR 方案 | Snap AR、ZOZO | 侧重虚拟量体，非真实试穿 |
| 开源项目 | OOTDiffusion、IDM-VTON | 单点方案，不成体系，缺交付能力 |

## Risks

| 风险 | 影响 | 缓解 |
|------|------|------|
| LHM/FASHN VTON 学术许可 | 商用受限 | v1 前确认各项目 LICENSE 和商用边界，必要时切换 Apache/MIT 替代方案 |
| 2D 试穿到 3D 模型纹理映射 | 360° 查看效果差 | 重点工程攻坚，UV 展开 + 多视角补全 |
| 工程整合壁垒薄 | 竞品可复制 | 加速交付 + 场景深度绑定客户，用切换成本锁客 |
| 手机端 3D 渲染性能 | 移动端体验差 | 模型压缩 + LOD + 中端机 ≥ 25FPS 为硬指标 |

## Vision

如果 v1 跑通 3-5 个客户，v2 扩展方向：
- **直播实时试穿**：抖音/快手直播中，观众一键生成数字分身实时试穿主播展示的服装
- **AI 尺码推荐**：根据数字人体型 + 各国尺码对照表，推荐最佳尺码（跨境场景 killer feature）
- **社交试穿分享**：试穿效果图/短视频一键分享小红书/朋友圈，UGC 种草
- 配饰试穿（眼镜/帽子/包包）+ 商品走秀视频生成

长期目标：成为电商虚拟试穿领域的"WordPress"——开源基础版免费提供，商业交付版 + 定制服务收费。
